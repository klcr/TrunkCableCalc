```markdown
# 制約条件（Constraints）

実装過程で発見・決定された制約条件を記録するディレクトリ。

## 目的

- 設計判断とその根拠をトレーサブルにする
- 将来の開発者（人間・AI 問わず）が「なぜこうなっているか」を理解できるようにする
- 暫定措置と恒久対応の区別を明確にする

## ファイル命名規約

`{連番3桁}-{kebab-case の概要}.md`

例: `001-repository-di-pattern.md`

## テンプレート

各制約条件ドキュメントには以下のセクションを含める:

```markdown
# 制約条件 NNN: タイトル

## 発生日

YYYY-MM-DD

## 対象レイヤー

影響を受けるレイヤー・パッケージ

## 制約の概要

1〜2文で制約を要約

## 背景

なぜこの制約が生じたか

## 制約詳細

具体的な制約内容（箇条書き）

## 影響範囲

影響を受けるファイル・モジュール

## 今後の対応

制約の解消・緩和に向けたアクション
```

| No. | タイトル | 対象 | 状態 |
