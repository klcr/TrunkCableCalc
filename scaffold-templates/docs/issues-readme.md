# 事象管理 README テンプレート

以下のテンプレートを `{{PROJECT_NAME}}/docs/issues/README.md` として生成してください。
同時に `{{PROJECT_NAME}}/docs/issues/reports/` ディレクトリも作成してください（空の `.gitkeep` を配置）。

---

```markdown
# 事象管理（Issues）

実装中に発生したブロッカーや課題を記録し、進捗と解決を追跡するディレクトリ。

## 目的

- 実装の詰まり（ブロッカー）を可視化し、対応状況を共有する
- 調査・対応の経緯を時系列で残し、再発時に参照できるようにする
- 解決の判定はユーザーが行う（自動解決しない）

## ファイル命名規約

`reports/{連番3桁}-{kebab-case の概要}.md`

例: `reports/001-build-error-on-ci.md`

## 状態の定義

| 状態 | 意味 |
| --- | --- |
| 未解決 | 調査中・対応中。進捗は随時追記 |
| 解決済 | ユーザーが解決を確認した |

## テンプレート

各事象レポートには以下のセクションを含める:

```markdown
# 事象 NNN: タイトル

## 起票日

YYYY-MM-DD

## 対象レイヤー

影響を受けるパッケージ

## 事象の概要

1〜2文で要約

## 発生状況

何をしようとして、どこで詰まったか

## 調査・対応の経緯

- YYYY-MM-DD: 対応内容（時系列で追記）

## 関連ドキュメント

関連する制約条件や ADR へのリンク

## 解決

（未解決の場合は「未解決」と記載。ユーザーが解決を確認した場合に記入）

- 解決日: YYYY-MM-DD
- 解決方法: どのように解決したか
```

## 一覧

| No. | タイトル | 対象 | 状態 | 起票日 |
| --- | --- | --- | --- | --- |
```
