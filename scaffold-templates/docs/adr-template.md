# ADR テンプレート

以下のテンプレートを `{{PROJECT_NAME}}/docs/adr/` ディレクトリに配置してください。
初期状態では ADR ディレクトリだけ作成し、テンプレートは参考情報として提供します。

## ADR のファイル命名規約

`{連番3桁}-{kebab-case の概要}.md`

例: `001-monorepo.md`, `002-database-choice.md`

## テンプレート

新しい ADR を作成する際は以下の形式に従ってください:

```markdown
# ADR NNN: タイトル

## ステータス

提案 / 承認 / 廃止

## コンテキスト

この決定が必要になった背景・状況

## 決定

何を決定したか

## 理由

なぜこの決定をしたか（検討した代替案と比較を含む）

## 結果

この決定によって生じる影響（ポジティブ・ネガティブ両方）
```

## 初期 ADR の例

プロジェクト開始時に以下のような ADR を記録することを推奨します:

- `001-monorepo.md` — モノレポ構成の採用理由
- `002-{database}.md` — データベース選定理由
- `003-{framework}.md` — 主要フレームワーク選定理由
